## Run

```sh
# install
$ npm i

# dev - http://localhost:3001/flutter_tilt_book/
$ npm run dev

# build (/out)
$ npm run build
```

## Edit document

The document menu can be edited in [`./settings/documents.ts`](./settings/documents.ts).

The document content directory is located at [`./contents/[locale]/`](./contents/).  
Documents are named `index.mdx` and written using [`Markdown`](https://github.github.com/gfm/).  
The following document metadata is supported:  

```md
---
title: Changelog
description: xxxxxx
keywords: ["changelog", "xxx"]
---
```

Custom components are also supported, for example:

```md
<Note title="Important" type="success">
xxxxxx
</Note>
```

You can find the list of supported components in [`./lib/components.ts`](./lib/components.ts).

## i18n

i18n Configuration: [`./settings/i18n.ts`](./settings/i18n.ts)  
Website dictionary directory: [`./i18n/dictionaries/`](./i18n/dictionaries/)  
Document content: [`./contents/[locale]/`](./contents/)  

If you need to add a new locale, configure the corresponding locale in `./settings/i18n.ts` and in the `searchData` of `./lib/search-data-utils.ts`.  
For documents, create the corresponding structured content in `./contents/[locale]/`.

## Directory structure

```md
├── app/                 # Next.js application source files
│   ├── [locale]/        # Locale-specific pages and docs
│   │   ├── docs/        # Locale documentation
│   │   ├── layout.tsx   # Locale layout
│   │   └── page.tsx     # Locale main page
│   ├── error.tsx        # Error page
│   ├── layout.tsx       # Root layout
│   ├── not-found.tsx    # 404 page
│   ├── page.tsx         # Main page
│   ├── robots.ts        # Robots.txt generator
│   └── sitemap.ts       # Sitemap generator
├── components/          # Reusable UI components
│   ├── markdown/        # Markdown-related components
│   ├── navigation/      # Navigation components
│   ├── providers/       # Context providers
│   └── ui/              # UI elements
├── contents/            # Documentation content for each locale
├── dist/                # Build output directory (generated by command, see `tsconfig.scripts.json`)
├── i18n/                # Internationalization utilities and dictionaries
├── lib/                 # Library utilities and helpers
├── public/              # Static assets (images, search data, etc.)
├── scripts/             # Scripts
├── settings/            # Project settings and configuration
├── styles/              # Global and component styles
├── tools/               # Build and data generation tools
...
```
